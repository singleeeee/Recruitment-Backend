## 1. 项目概述

### 1.1 项目背景
随着高校社团文化的蓬勃发展，传统线下招新方式存在信息传递效率低、管理流程繁琐、人力成本高、数据处理困难等问题。为解决这些痛点，本项目旨在开发一个智能化在线招新系统，实现招新流程数字化、自动化、智能化。

### 1.2 项目目标
- 实现招新全流程在线化，包括后台管理、候选人投递简历、后续面试安排等等，以提升管理和招新效率
- 通过AI辅助挑选简历并个性化生成题库来提高招新质量和工作效率
- 提供多端兼容的响应式用户体验，候选人和管理员使用同一个网站，根据登录账号的角色实现权限管理
- 建立安全可靠的数据管理体系

### 1.3 目标用户群体
1. **社团管理员**：负责创建招新活动、审核申请、安排面试、发送通知等
2. **候选人**：在校学生，浏览社团信息、招新信息、提交申请、查看进度
3. **系统管理员**：管理系统用户、权限配置、数据维护

## 3. 功能性需求

### 3.1 用户管理模块
#### 3.1.1 用户注册与登录
- **功能描述**：支持邮箱注册和登录
- **输入**：邮箱、密码
- **处理**：
  1. 验证邮箱格式和唯一性
  2. 密码加密存储（bcrypt）
  4. JWT令牌生成和验证
- **输出**：登录成功返回JWT令牌，失败返回错误信息

#### 3.1.2 用户角色与权限
| 角色       | 权限描述                                           |
| ---------- | -------------------------------------------------- |
| 候选人     | 查看社团介绍、提交申请、查看简历状态、修改个人信息 |
| 社团管理员 | 创建招新活动、审核申请、安排面试、发送通知         |
| 系统管理员 | 用户管理、系统配置、数据导出                       |

#### 3.1.3 个人信息管理
- 修改个人资料（姓名、学号、学院、专业等）
- 上传头像
- 修改密码
- 绑定邮箱和手机

### 3.2 招新管理模块
#### 3.2.1 招新批次管理
```typescript
// 招新批次数据结构
interface RecruitmentBatch {
  id: string;
  title: string;          // 招新标题
  clubId: string;         // 所属社团
  description: string;    // 招新描述

#### 3.2.2 招新活动流程
1. **创建招新**：管理员设置招新基本信息、时间、要求
2. **发布招新**：候选人可查看和申请
3. **申请阶段**：候选人提交申请材料
4. **筛选阶段**：AI初筛 + 人工审核
5. **面试阶段**：安排面试、发送通知
6. **录用阶段**：发送录用通知
7. **归档阶段**：数据归档和统计

```

```

    "scoreBreakdown": {
      "skillMatch": 90,
      "experience": 80,
      "presentation": 85,
      "potential": 75

  }
  ```

## 5. 数据库设计

### 5.1 核心实体关系
```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(50) NOT NULL DEFAULT 'candidate',
    -- 个人信息字段
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 招新批次表
CREATE TABLE recruitment_batches (
    id UUID PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    club_id UUID NOT NULL,
    description TEXT,
    start_time TIMESTAMP NOT NULL,
    end_time TIMESTAMP NOT NULL,
    status VARCHAR(50) DEFAULT 'draft',
    settings JSONB DEFAULT '{}',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 申请表
CREATE TABLE applications (
    id UUID PRIMARY KEY,
    recruitment_id UUID NOT NULL,
    applicant_id UUID NOT NULL,
    status VARCHAR(50) DEFAULT 'draft',
    resume_text TEXT,
    ai_score DECIMAL(5,2),
    ai_analysis JSONB DEFAULT '{}',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 面试安排表
CREATE TABLE interviews (
    id UUID PRIMARY KEY,
    application_id UUID NOT NULL,
    scheduled_time TIMESTAMP NOT NULL,
    interviewers UUID[] NOT NULL,
    status VARCHAR(50) DEFAULT 'scheduled',
    feedback JSONB DEFAULT '{}'
);

-- 文件表
CREATE TABLE files (
    id UUID PRIMARY KEY,
    filename VARCHAR(255) NOT NULL,
    original_name VARCHAR(255) NOT NULL,
    mime_type VARCHAR(100) NOT NULL,
    size BIGINT NOT NULL,
    storage_path VARCHAR(500) NOT NULL,
    uploaded_by UUID NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 9. 部署运维

### 9.1 部署环境
```yaml
# docker-compose.yml 示例
version: '3.8'
services:
  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_DB: recruitment
      POSTGRES_USER: admin
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    
  redis:
    image: redis:7-alpine
    
  backend:
    build: ./backend
    depends_on:
      - postgres
      - redis
    environment:
      DATABASE_URL: postgresql://admin:${DB_PASSWORD}@postgres:5432/recruitment
      
  frontend:
    build: ./frontend
    ports:
      - "3000:3000"
```
